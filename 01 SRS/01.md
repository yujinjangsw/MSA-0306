요청하신 비즈니스 기능, 비기능 요구사항, 도메인 이벤트 및 액터, 그리고 MSA(Microservices Architecture) 경계를 아래와 같이 도출했습니다.

### 1. 기능 요구사항 목록 (Functional Requirements)

시스템의 핵심 비즈니스 기능을 식별하고, 비즈니스 가치와 구현 시급성을 기준으로 우선순위(P1: 매우 높음, P2: 높음, P3: 보통)를 부여했습니다.

| 도메인 | 기능명 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| **영화 관리** | 영화 등록 (Create) | 관리자가 새로운 영화 정보(제목, 장르, 감독, 개봉일 등)를 등록합니다. | **P1** |
|  | 영화 조회 (Read) | 사용자가 영화의 상세 정보를 조회합니다. | **P1** |
|  | 영화 수정 (Update) | 관리자가 기존 영화 정보를 수정합니다. | **P2** |
|  | 영화 삭제 (Delete) | 관리자가 영화 정보를 삭제하거나 숨김 처리합니다. | **P2** |
| **리뷰/평점** | 리뷰 및 평점 등록 | 일반 사용자가 영화에 대한 리뷰 텍스트와 별점을 남깁니다. | **P1** |
|  | 리뷰 수정/삭제 | 본인이 작성한 리뷰를 수정하거나 삭제합니다. | **P2** |
|  | 평점 집계 | 영화별 평균 평점을 계산하여 영화 정보에 반영합니다. | **P1** |
| **검색 및 탐색** | 기본 검색 | 제목, 감독, 배우, 장르 등의 키워드로 영화를 검색합니다. | **P1** |
|  | 필터링 및 정렬 | 개봉일, 평점순 등으로 검색 결과를 필터링하고 정렬합니다. | **P2** |
| **사용자 관리** | 회원가입 및 로그인 | 시스템 사용을 위한 계정을 생성하고 인증합니다. | **P1** |
|  | 마이페이지 | 사용자가 찜한 영화 목록과 작성한 리뷰 모아보기를 제공합니다. | **P3** |

### 2. 비기능 요구사항 목록 (Non-Functional Requirements)

안정적이고 지속 가능한 서비스를 위해 다음과 같은 시스템 품질 속성을 정의합니다.

* **확장성 (Scalability):**
* 특정 영화(예: 천만 관객 돌파 화제작)에 대한 트래픽 급증 시, 조회 및 검색 서비스가 독립적으로 Auto-scaling(자동 확장)되어야 합니다.
* 데이터베이스는 읽기 트래픽 분산을 위해 Read Replica를 구성합니다.


* **내결함성 (Fault Tolerance):**
* 리뷰 서비스나 추천 서비스에 장애가 발생하더라도, 핵심 기능인 '영화 조회 및 검색'은 정상적으로 작동해야 합니다. (회로 차단기/Circuit Breaker 패턴 적용)


* **보안 (Security):**
* 모든 API 통신은 HTTPS를 통해 암호화되어야 합니다.
* JWT(JSON Web Token)를 이용해 사용자와 관리자의 권한(Role-Based Access Control)을 철저히 분리합니다.
* 사용자의 비밀번호는 단방향 암호화(예: bcrypt)하여 저장합니다.


* **응답 시간 SLA (Service Level Agreement):**
* 영화 검색 및 상세 정보 조회 API의 95%는 **200ms 이내**에 응답해야 합니다.
* 리뷰 작성 및 평점 업데이트 등 쓰기 작업은 **500ms 이내**에 처리되어야 합니다.

### 3. 주요 액터 및 도메인 이벤트

시스템과 상호작용하는 주요 액터와, 시스템 상태 변화를 나타내는 핵심 도메인 이벤트입니다.

**주요 액터 (Actors):**

1. **관리자 (Admin):** 영화 메타데이터를 관리하고 시스템 전반의 컨텐츠를 통제하는 주체.
2. **일반 사용자 (Regular User):** 영화를 검색 및 조회하고, 자신의 의견(리뷰/평점)을 생성 및 소비하는 주체.

**도메인 이벤트 (Domain Events):**

* `MovieRegistered` (영화 등록됨)
* `MovieUpdated` (영화 정보 수정됨)
* `ReviewPosted` (새로운 리뷰가 작성됨) $\rightarrow$ *평점 집계 서비스 트리거*
* `ReviewDeleted` (리뷰가 삭제됨)
* `AverageRatingRecalculated` (영화 평균 평점이 재계산됨) $\rightarrow$ *영화 검색/조회 모델 업데이트 트리거*
* `UserRegistered` (신규 사용자 가입됨)

### 4. MSA 경계 정의 (Bounded Contexts)

도메인 주도 설계(DDD)를 바탕으로 마이크로서비스의 경계를 다음과 같이 명확히 나눕니다.

1. **Movie Catalog Service (영화 카탈로그 서비스):**
* **역할:** 영화의 기본 메타데이터(제목, 감독, 출연진 등)의 원본(Source of Truth)을 관리합니다.
* **주요 기능:** 영화 등록, 수정, 삭제 (관리자 전용)


2. **Review & Rating Service (리뷰/평점 서비스):**
* **역할:** 사용자가 남기는 평가 데이터를 독점적으로 처리합니다.
* **주요 기능:** 리뷰 작성/수정/삭제, 영화별 평점 로직 계산.


3. **Search & Discovery Service (검색 서비스):**
* **역할:** 빠른 조회와 복잡한 검색 쿼리를 담당합니다. (CQRS 패턴 적용)
* **주요 기능:** 키워드 검색, 필터링. 카탈로그와 리뷰 서비스에서 이벤트를 구독하여 Elasticsearch 등의 검색 엔진 데이터를 동기화합니다.


4. **User Service (사용자 서비스):**
* **역할:** 사용자 정보 및 인증/인가를 담당합니다.
* **주요 기능:** 회원가입, 로그인, JWT 발급, 프로필 관리.

### 5. 간략한 유스케이스 다이어그램

**[텍스트 기반 유스케이스 요약]**

* **일반 사용자 (Regular User)**
* $\longrightarrow$ 영화 검색 및 조회
* $\longrightarrow$ 리뷰 작성 / 수정 / 삭제
* $\longrightarrow$ 영화 평점 부여
* $\longrightarrow$ 회원가입 및 로그인


* **관리자 (Admin)**
* $\longrightarrow$ 영화 정보 등록 / 수정 / 삭제
* $\longrightarrow$ 부적절한 리뷰 관리 (숨김 처리)
* $\longrightarrow$ *일반 사용자의 모든 기능(조회 등) 포함*
